# Contributing to LLMR

Thank you for your interest in LLMR.

## Reporting bugs

Please open a GitHub issue and include:

- the provider and model you used
- a minimal reproducible example
- the full error message
- your R version and
  [`sessionInfo()`](https://rdrr.io/r/utils/sessionInfo.html) output
- whether the problem occurs with the CRAN release or the development
  version

## Requesting support

Use GitHub issues for questions about installation, configuration, or
package behavior. Please include enough detail for the problem to be
reproduced.

## Suggesting features

Open an issue describing the research use case, the desired behavior,
and why the current interface is insufficient.

## Pull requests

Please discuss substantial interface or behavior changes in an issue
before opening a pull request. Keep pull requests focused. When code
changes are proposed, run the test suite and package checks before
submitting.
